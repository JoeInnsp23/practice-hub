## Quality Framework

### Non-Negotiable Gates ğŸ”’

**Zeus enforces these absolutely:**

1. **Brief Validation Gate**
   - Cannot proceed to PRD until brief validated
   - Completeness, clarity, consistency checked

2. **PRD Validation Gate**
   - Cannot proceed to TDD until PRD validated
   - All sections present, technically feasible

3. **TDD Validation Gate**
   - Cannot proceed to epics until TDD validated
   - Phases logical, stories implementable

4. **Pre-Story Quality Gate**
   - Cannot start story until pnpm format/lint/typecheck pass
   - ALL issues fixed (even pre-existing)

5. **QA Gate**
   - Cannot proceed to next story until Apollo says PASS
   - 90% coverage, security validated, performance checked

6. **Epic Completion Gate**
   - Cannot merge until all stories DONE
   - All QA gates passed

7. **Feature Completion Gate**
   - Cannot merge to main until all epics DONE
   - Documentation synchronized

### Quality Standards

**Code:**
- 90% minimum Vitest test coverage
- Zero lint errors
- Zero format issues
- Zero type errors
- No placeholders or TODOs

**Security (PARAMOUNT):**
- Multi-tenant isolation validated on EVERY QA pass
- Staff/client access separation verified
- Cross-tenant access tests required
- Reference: `/docs/architecture/multi-tenancy.md`

**Performance:**
- Page load < 3 seconds
- API response < 500ms
- No N+1 queries
- Query efficiency validated

**Documentation:**
- Drift detected automatically
- Auto-synchronized after QA
- Project status tracked
